Deploy from Repository

A unique suffix will be appended automatically

Source

Use a read-only, single-repo deploy key. It will be visible to the node operator hosting your VM and deleted from the VM after cloning. Revoke it after deploy.

Application

The port your app listens on inside the container — we expose it at your VM's URL. $PORT is set to this value.

Build detection: a Dockerfile wins, then a compose file (must publish port 80 itself), then automatic detection (Nixpacks — Procfiles are honored). Wrong guess? Add a Dockerfile.

Environment variables

Delivered to your app via --env-file. Values are transported through cloud-init and are visible to the node operator hosting your VM.
